# M3U8 下载器：开发规划文档

## 1. 需求分析

### 1.1 核心功能

1. **M3U8 解析**：解析顶级 M3U8 (Master Playlist) 和媒体 M3U8 (Media Playlist)。
2. **异步下载**：支持多线程并发下载 `.ts` 视频分片。
3. **断点续传**：支持断点续传，已下载的 `.ts` 分片不重新下载，以m3u8Playlist的Hash作为唯一标识符。
4. **解密支持**：处理 AES-128 加密的分片（自动获取 Key）。
5. **视频合并**：将下载的 `.ts` 分片按顺序合并，并转换为 `.mp4` 格式。
6. **GUI 交互**：基于 `egui` 提供直观的任务进度显示、URL 输入和保存路径选择。

### 1.2 非功能性要求

- **鲁棒性**：处理网络抖动、重试机制。
- **性能**：并行下载以最大化利用带宽。
- **极致审美**：UI 界面简洁现代。
- **配置文件**：支持配置文件，设置并发下载数等配置项。

---

## 2. 系统架构

### 2.1 模块划分

- `m3u8-downloader-core`: 核心逻辑库。
  - `parser`: M3U8 解析逻辑。
  - `downloader`: 异步分片下载器 (Tokio + Reqwest)。
  - `crypto`: AES 解密处理器。
  - `merger`: 视频流合并与 FFmpeg 封装。
- `m3u8-downloader-gui`: 基于 `egui` 的前端展示。

### 2.2 流程设计

1. 用户输入 URL → `parser` 获取播放列表。
2. `downloader` 建立下载任务队列。
3. 多协程并行下载分片，若检测到 `EXT-X-KEY` 则调用 `crypto` 解密。
4. 下载完成后，`merger` 按序号合并文件。
5. 输出最终的 MP4 文件。

---

## 3. 技术选型

- **编程语言**：Rust (保证内存安全与高性能)
- **UI 框架**：`egui` (轻量级、跨平台、高性能)
- **异步运行时**：`tokio`
- **网络请求**：`reqwest`
- **M3U8 解析**：`m3u8-rs`
- **解密库**：`aes`, `cbc`
- **进度条/日志**：`indicatif` (CLI) / `egui` components (GUI)
